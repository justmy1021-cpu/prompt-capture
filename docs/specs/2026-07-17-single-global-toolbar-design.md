# Prompt Capture 全局单实例工具栏设计

## 方案结论

Prompt Capture 在整个 Chrome 中只显示一个逻辑实例：仅当前聚焦 Chrome 窗口的活动标签页显示工具栏，其他标签页和窗口全部隐藏。由于 Chrome 不允许同一个网页 DOM 跨标签页移动，各页面仍保留独立载体，但显示目标、位置和分析会话均由扩展后台统一控制，用户感知为同一个插件。

本文替代 `2026-07-17-global-toolbar-selection-cancel-design.md` 中“所有支持网页同时显示工具栏”的设计；原文关于全局启停、历史或设置取消选择的要求继续有效。

## 目标

- 整个 Chrome 同一时间最多显示一个 Prompt Capture 工具栏。
- 切换标签页或 Chrome 窗口时，工具栏移动到最后聚焦窗口的活动标签页。
- 工具栏位置、当前页面、生成进度、结果和错误状态跨标签页统一。
- 尚未确认的网页选择在显示目标变化时取消。
- 已确认的模型生成在后台继续，不因切换标签页或关闭来源页面中断。
- 全局启停状态、位置、设置和历史在浏览器重启后保留。

## 非目标

- 不尝试把同一个 iframe 或 DOM 节点跨标签页搬移。
- 不改成 Chrome 侧边栏或独立窗口。
- 不让选择图片、当前页面候选或框选遮罩跨页面继续。
- 不保证完全退出 Chrome 后恢复正在进行的模型请求或临时分析页面。
- 不改变版本号 `0.2.5`、模型协议、历史记录结构或现有视觉样式。

## 唯一显示目标

后台维护最后一次获得焦点的 Chrome 窗口，并把该窗口的活动标签页作为唯一显示目标。

触发重新计算的事件包括：

- 点击扩展图标开启或关闭工具栏。
- `chrome.tabs.onActivated`：切换活动标签页。
- `chrome.windows.onFocusChanged`：切换 Chrome 窗口。
- `chrome.tabs.onUpdated`：活动标签页刷新或跳转完成。
- `chrome.tabs.onRemoved`：显示目标被关闭。
- 内容脚本启动后主动查询当前页面是否为显示目标。

当 Chrome 暂时失去系统焦点并收到 `WINDOW_ID_NONE` 时，保留最后一个 Chrome 显示目标，避免切换到其他 App 后工具栏突然消失。

如果当前活动标签页是 `chrome://`、扩展管理页或其他不支持注入的页面，所有网页暂时隐藏工具栏；切回普通网页后自动恢复。

## 串行调度与消息

所有显示目标重算进入同一个串行队列。每次重算读取最新的全局启停状态、最后聚焦窗口和活动标签页，然后：

1. 向目标标签页发送明确显示消息。
2. 向其他支持页面发送明确隐藏消息。
3. 旧目标隐藏时先取消该页面尚未确认的采集选择。

快速连续切换时，队列最终以最后一次焦点状态为准，不允许两个页面同时保持显示。

内容脚本不再仅凭 `promptCaptureToolbarEnabled` 自行显示。启动时必须向后台查询，只有后台确认当前标签页是唯一目标时才显示。

## 全局位置

位置继续保存在 `chrome.storage.local` 的 `promptCaptureToolbarPosition`：

- 拖动结束后只写入一次。
- 后台发送显示消息时附带最新位置。
- 内容脚本先应用位置，再显示工具栏，避免切换页面时从旧位置跳动。
- 每个页面根据自己的视口尺寸把位置限制在可见范围内；较小页面只做本地收缩，不回写覆盖原始全局位置。

因此只有一个页面可见，不需要拖动过程中的跨页面高频同步。

## 全局分析会话

临时分析状态保存在 `chrome.storage.session` 的 `promptCaptureActiveSession`。该存储在 Chrome 运行期间跨标签页共享，完全退出 Chrome 后自动清空。

会话包含：

- `phase`：`idle`、`required`、`generating`、`result`、`error`、`history` 或 `settings`。
- `previousPhase`：从历史或设置返回时使用。
- `activePrompt`：图片、风格或布局 Prompt。
- `capture`：当前截图、选择类型和来源页面。
- `record`：本次成功生成的完整结果。
- `error`：失败提示。
- `updatedAt`：解决多个实例更新顺序。

选择图片、当前页面候选和框选阶段仍由当前网页内容脚本本地管理，不写入全局会话。切换标签页或窗口时，旧页面收到隐藏消息并取消选择，新页面恢复切换前的稳定阶段。

## 生成流程

用户确认候选后，后台成为生成生命周期的权威来源：

1. 收到截图后先把全局会话更新为 `generating`。
2. 模型请求在后台继续执行。
3. 成功后写入历史记录并把会话更新为 `result`。
4. 失败后把会话更新为 `error`，保留可重试截图和错误信息。
5. 所有工具栏实例监听 `chrome.storage.onChanged` 的 `session` 区域，并恢复相同界面。

即使来源标签页被隐藏或关闭，后台请求仍继续；新活动页面显示生成进度和最终结果。

如果截图过大导致完整会话写入超过临时存储限制，后台降级为不含截图数据的生成状态。模型请求继续，界面使用默认预览；生成成功后优先通过历史记录中的结果恢复。

## 界面导航同步

进入历史或设置时：

1. 取消当前页面尚未确认的选择。
2. 更新全局会话的 `phase` 和 `previousPhase`。
3. 其他隐藏实例通过会话变更同步状态。

切换页面后，新工具栏继续显示历史、设置、生成中、结果或错误页面，而不是回到默认页。Prompt 类型切换也写入全局会话。

## 关闭与重新开启

- 点击扩展图标关闭或点击工具栏关闭按钮：写入全局关闭状态并隐藏全部页面。
- 关闭不清除当前 `chrome.storage.session` 会话。
- 同一 Chrome 运行期间重新开启时，恢复上次界面、结果和位置。
- 完全退出 Chrome 后，重新开启恢复启停状态、位置、设置与历史，但临时分析界面回到默认状态。

## 异常处理

- 单个页面发送或注入失败不影响其他页面。
- 不支持页面不显示错误提示。
- 内容脚本收到重复显示、隐藏或取消消息时保持幂等。
- 目标页面刷新期间先隐藏旧实例，页面重新就绪后主动查询并恢复。
- 来源标签页关闭后，已提交的模型请求继续；未确认选择直接丢弃。
- 会话更新按 `updatedAt` 接受较新状态，避免隐藏实例用旧状态覆盖新页面。

## 测试与验收

### 自动化测试

- 焦点窗口和活动标签页只产生一个显示目标。
- `WINDOW_ID_NONE` 保留最后 Chrome 窗口。
- 标签页切换、窗口切换、刷新和关闭进入串行重算。
- 不支持页面导致全部隐藏，切回普通网页恢复。
- 显示消息附带最新全局位置。
- 内容脚本启动时向后台查询唯一显示资格。
- 隐藏旧目标会取消网页选择。
- `generating`、`result`、`error`、历史、设置和 Prompt 类型同步到会话。
- 来源页面隐藏或关闭后，后台仍完成生成并保存历史。
- 保留并通过现有 31 项测试。

### 手动验收

1. 打开多个标签页和多个 Chrome 窗口，始终只有最后聚焦窗口的活动标签页显示插件。
2. 切换标签页和窗口，工具栏保持同一位置。
3. 在选择图片、当前页面或框选阶段切换页面，旧页面遮罩消失。
4. 确认生成后切换标签页或窗口，新页面继续显示生成进度并展示结果。
5. 在结果、历史或设置页切换标签页，新页面保持相同页面。
6. 打开 `chrome://extensions` 时插件暂时消失，切回普通网页后恢复。
7. 点击任一关闭入口后全部隐藏，重新开启恢复同一 Chrome 会话中的状态。
