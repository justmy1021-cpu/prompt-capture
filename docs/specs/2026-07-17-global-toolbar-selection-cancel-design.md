# Prompt Capture 全局显示与采集取消设计

## 背景

当前工具栏的显示状态只存在于单个网页。用户在一个网页中打开插件后，切换到其他网页仍需再次点击扩展图标。同时，用户启动“当前页面”“选择图片”或“框选截图”后，点击历史或设置入口不会主动结束网页上的选择状态。

本次改动解决两个问题：

1. 用户开启一次插件后，工具栏在所有支持的网页中保持显示。
2. 用户在采集过程中进入历史或设置时，立即取消当前选择并恢复网页状态。

## 目标

- 提供统一、可持久化的工具栏全局启停状态。
- 已打开网页、新打开网页、刷新页面和网址跳转后均遵循同一状态。
- 任一工具栏的关闭按钮可以全局关闭工具栏。
- 历史和设置入口统一取消未确认的采集选择。
- 不改变已确认并进入模型生成阶段的任务。

## 非目标

- 不支持按网站或按标签页单独保存显示状态。
- 不在 Chrome 设置页、扩展页等禁止内容脚本运行的页面显示工具栏。
- 不增加模型生成请求的中断能力。
- 不改变现有采集、生成、历史记录和模型设置的数据结构。

## 全局显示架构

### 全局状态

在 `chrome.storage.local` 中增加布尔字段 `promptCaptureToolbarEnabled`：

- `true`：所有支持的网页显示工具栏。
- `false` 或字段不存在：所有网页隐藏工具栏。

默认关闭，避免用户安装或升级插件后未经操作就在所有网页上自动显示。

### 开启与关闭

点击浏览器工具栏中的 Prompt Capture 扩展图标时，由后台读取并反转全局状态：

1. 写入新的全局状态。
2. 查询所有已打开标签页。
3. 向支持的网页发送明确的显示或隐藏消息。
4. 消息发送失败时，尝试注入当前版本的内容脚本后重发；无法注入的系统页面直接跳过。

网页内工具栏的关闭按钮不再只隐藏当前网页，而是向后台发送全局关闭请求。后台写入 `false` 并向所有支持的网页广播隐藏消息。

### 页面启动同步

内容脚本启动后读取 `promptCaptureToolbarEnabled`：

- 状态为 `true` 时创建并显示工具栏。
- 状态为 `false` 或不存在时保持隐藏。

因此，新建标签页、刷新网页、同标签页网址跳转和浏览器重启后，都能恢复一致状态。

### 消息边界

后台负责：

- 保存全局状态。
- 处理扩展图标点击与网页内全局关闭请求。
- 向所有标签页广播明确的显示或隐藏消息。

内容脚本负责：

- 根据后台消息显示或隐藏当前网页的工具栏。
- 启动时读取全局状态。
- 隐藏工具栏前清除当前网页的采集选择。

嵌入式工具栏界面负责：

- 将关闭动作转发为全局关闭请求。
- 在进入历史或设置前，重置界面内的选择状态并通知内容脚本取消网页选择。

## 采集取消设计

### 触发范围

用户启动以下任一操作后，点击历史或设置入口均触发取消：

- 当前页面
- 选择图片
- 框选截图

取消覆盖正在选择、正在生成当前页面候选、已经形成候选但尚未确认的状态。

### 取消顺序

历史和设置入口共用一个导航方法，按以下顺序执行：

1. 清空工具栏内的候选截图、选择模式、重试数据、错误信息和选择提示。
2. 向内容脚本发送取消选择消息。
3. 内容脚本移除图片描边、截图遮罩、页面描边及相关事件监听。
4. 内容脚本解除滚动和键盘锁定。
5. 工具栏进入历史或设置页面。

该顺序确保用户看到子页面时，网页已经恢复正常交互。

### 异步竞态

“当前页面”会异步截取候选图。取消时将当前选择对象清空；截图返回后必须再次检查选择仍然有效，失效结果不得重新显示候选或改变工具栏页面。

如果用户已经确认候选并进入模型生成阶段，本次导航只清理选择界面，不取消已经发出的模型请求。生成结果仍可写入历史记录。

## 异常处理

- `chrome://`、扩展管理页和其他禁止注入的页面直接跳过，不提示失败。
- 单个标签页发送或注入失败不影响其他标签页。
- 全局关闭必须先保存状态，再广播隐藏，避免新页面在广播期间重新显示。
- 连续快速点击扩展图标时，状态更新串行执行，以最后一次点击的结果为准。
- 内容脚本收到重复显示、隐藏或取消消息时保持幂等，不创建重复工具栏或残留遮罩。
- 全局隐藏沿用现有清理能力，确保结束拖动、清除选择并恢复页面交互。

## 测试与验收

### 自动化测试

- 全局状态键、后台切换逻辑和显式显示/隐藏消息保持一致。
- 后台广播覆盖所有标签页，并跳过不支持页面的失败。
- 内容脚本启动时读取全局状态。
- 网页内关闭按钮发送全局关闭请求。
- 历史和设置入口均通过统一路径发送取消选择消息。
- 图片选择、框选截图和当前页面模式取消后清除候选与选择状态。
- 当前页面截图异步返回时，不恢复已取消的候选。
- 保留并通过现有全部测试。

### 手动验收

1. 在网页 A 点击扩展图标，网页 A 和已打开的网页 B 均显示工具栏。
2. 新建网页 C、刷新网页 B、在网页 A 跳转网址，工具栏继续显示。
3. 在网页 B 点击工具栏关闭，网页 A、B、C 的工具栏全部隐藏。
4. 浏览器重启后保持上次的开启或关闭状态。
5. 分别进入选择图片、框选截图和当前页面模式，再点击历史，网页选择状态立即消失。
6. 重复上一步并点击设置，行为与历史一致。
7. 取消后网页可滚动、可点击、无图片描边或截图遮罩残留。
8. 在 Chrome 设置页等不支持页面操作时，不出现错误且不影响普通网页。

## 兼容性与发布

- 保持 Manifest V3、现有权限和本地存储方式不变。
- 新增状态字段不影响旧设置；字段不存在时按关闭处理。
- 内容脚本消息通道和版本标识同步升级，确保更新后已打开网页可替换旧实例。
- 实现完成后运行测试、生产构建，并检查安装包中不存在重复文件。
